Com base nas imagens, a sua planilha de formação de preço para a Nexo Brindes é bem estruturada e utiliza uma lógica de markup divisor com faixas de desconto por volume.

Aqui está o que identifiquei sobre a sua formação de preço:

1. Composição do Preço Base
O sistema soma os custos diretos e depois aplica uma carga de impostos, comissões e margem.

Custos Diretos: Custo do Produto + Personalização + Frete + Custos Extras (vistos na Página 1 e Planilha 3).

Carga Variável: Imposto Simples (ex: 9%) e Comissão (ex: 15%).

Margem de Lucro: Definida pela sua "Tabela de Porcentagem".

2. A Lógica de Quantidade e Desconto
Sim, é possível ver claramente na Página 2 e na Planilha 3 como a quantidade influencia o preço:

Escada de Margens: Na Página 2, você tem uma tabela onde, conforme a quantidade aumenta, a margem de lucro diminui:

Até 5.000 unidades: 45% de margem.

Até 8.000 unidades: 42%.

Acima de 10.000/15.000 unidades: 40%.

Preços Escalonados: Na Planilha 3, isso se reflete na lista abaixo do "Preço Venda Ideal" e "Preço Mínimo". Note que existem vários valores (R$ 8,59; R$ 8,03; R$ 7,86...), que representam o preço caindo à medida que o cliente compra volumes maiores e você aceita uma margem menor.

3. Diferença entre Preço Ideal e Mínimo
O Preço Ideal deve ser o que utiliza a margem cheia da tabela.

O Preço Mínimo (visto em azul na Planilha 3) parece ser o seu "limite de segurança", provavelmente calculado com uma margem de contribuição mínima para não ter prejuízo na operação.

4. Condições de Pagamento
Você também inseriu uma camada extra de desconto no fechamento:

À vista: -5% de desconto.

À vista sem impostos: -12% de desconto (provavelmente para faturamento direto ou situações específicas).

Resumindo: A tabela entende que, para vender grandes volumes (como os 13.400 da imagem), você reduz a margem de 45% para 40%, baixando o preço unitário para tornar o orçamento mais competitivo.

Python
def calcular_orcamento(custo_base, quantidade, personalizacao=0, frete=0):
    # 1. Somar custos diretos
    custo_total = custo_base + personalizacao + frete
    
    # 2. Definir a margem baseada na quantidade (conforme sua Tabela de Porcentagem)
    if quantidade <= 5000:
        margem = 0.45
    elif quantidade <= 8000:
        margem = 0.42
    else:
        margem = 0.40  # Margem mínima de lucro
        
    # 3. Taxas Fixas (conforme sua Planilha 1)
    imposto = 0.09
    comissao = 0.15
    
    # 4. Cálculo do Preço de Venda (Markup Divisor)
    # Soma das porcentagens que "saem" do preço:
    divisor = 1 - (imposto + comissao + margem)
    
    preco_unitario = custo_total / divisor
    faturamento_total = preco_unitario * quantidade
    
    return {
        "Preço Unitário": round(preco_unitario, 2),
        "Total Orçamento": round(faturamento_total, 2),
        "Margem Aplicada": f"{(margem*100)}%"
    }

# Exemplo de uso:
print(calcular_orcamento(custo_base=3.47, quantidade=13400, personalizacao=0.25))

Input da Quantidade: O usuário digita 900.

Cálculo Automático:

O sistema vê que 900 está na faixa de 45% de margem.

Ele soma Imposto (9%) + Comissão (15%) + Margem (45%) = 69%.

Ele faz a conta: R$ 5,90 / (1 - 0,69) → 5,90 / 0,31.

Resultado: O campo "Preço de Venda" no formulário é preenchido instantaneamente com R$ 19,03.

class MotorDePreco:
    def __init__(self):
        # Configurações fixas (conforme sua Planilha 1 e 3)
        self.imposto_simples = 0.09  # 9%
        self.comissao = 0.15        # 15%
        
        # Tabela de Margens (conforme sua Página 2)
        # Formato: (Até a quantidade, Margem)
        self.tabela_margens = [
            (5000, 0.45),
            (8000, 0.42),
            (15000, 0.40)
        ]

    def obter_margem_por_qtd(self, quantidade):
        # Busca a margem ideal baseada na quantidade
        for limite, margem in self.tabela_margens:
            if quantidade <= limite:
                return margem
        return 0.35  # Margem padrão para volumes gigantes acima de 15k

    def calcular(self, custo_produto, personalizacao, frete, quantidade, condicao_pagamento="padrao"):
        # 1. Soma dos Custos Diretos
        custo_total = custo_produto + personalizacao + frete
        
        # 2. Definição das Margens
        margem_ideal = self.obter_margem_por_qtd(quantidade)
        margem_minima = 0.20  # Exemplo: Você aceita cair até 20% de lucro no preço mínimo
        
        # 3. Cálculo do Markup Divisor (Preço = Custo / (1 - Taxas))
        # Preço Ideal
        divisor_ideal = 1 - (self.imposto_simples + self.comissao + margem_ideal)
        preco_ideal = custo_total / divisor_ideal
        
        # Preço Mínimo (usando a margem de segurança)
        divisor_minimo = 1 - (self.imposto_simples + self.comissao + margem_minima)
        preco_minimo = custo_total / divisor_minimo

        # 4. Aplicação de Descontos de Pagamento (conforme sua Planilha 3)
        if condicao_pagamento == "a_vista_5":
            preco_ideal *= 0.95
        elif condicao_pagamento == "a_vista_sem_imposto":
            # Aqui simulamos o desconto de 12% citado na sua tabela
            preco_ideal *= 0.88 

        return {
            "custo_direto": round(custo_total, 2),
            "margem_aplicada": f"{margem_ideal*100}%",
            "preco_venda_ideal": round(preco_ideal, 2),
            "preco_minimo_seguro": round(preco_minimo, 2),
            "faturamento_total": round(preco_ideal * quantidade, 2)
        }

# --- EXEMPLO DE USO NO REPLIT ---
motor = MotorDePreco()

# Simulando os dados que viriam do seu formulário
meu_pedido = motor.calcular(
    custo_produto=3.47, 
    personalizacao=0.25, 
    frete=0, 
    quantidade=13400
)

print(f"Sugestão de Preço: R$ {meu_pedido['preco_venda_ideal']}")
print(f"Preço Mínimo (Limite): R$ {meu_pedido['preco_minimo_seguro']}")
print(f"Total do Orçamento: R$ {meu_pedido['faturamento_total']}")
Como integrar isso ao seu formulário:
No Frontend (HTML/JS): Quando o usuário selecionar o produto e digitar a quantidade, você faz uma chamada para o seu backend no Replit passando esses valores.

No Backend: * O sistema busca no seu banco de dados (ou num dicionário simples) o custo_produto daquele item específico.

Ele executa a função calcular() acima.

Ele devolve para o formulário o Preço Venda Ideal (para o vendedor oferecer primeiro) e o Preço Mínimo (que fica escondido ou serve como trava de desconto).

O "Pulo do Gato" para o Preço Mínimo:
Note que no código eu usei uma margem_minima = 0.20. Na sua planilha, o "Preço Mínimo" parece ser calculado de forma a cobrir apenas os custos operacionais e uma pequena sobra. Você pode ajustar essa variável para o valor que você considera o "limite da negociação".

Dica para o Replit: Se você estiver usando Flask, você pode criar uma rota @app.route('/calcular-preco') que recebe os dados via POST e retorna esse JSON para o seu formulário.
